## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL searchParams -> Supabase .order() | Sort column name from URL could be used for SQL injection if passed directly |
| Client Component -> Server Action | Status transition value from UI could bypass state machine |
| Template form input -> DB write | Template body could contain malicious content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | lib/queries/leads.ts | mitigate | Validate sort column against SORTABLE_COLUMNS allowlist before passing to .order() — reject unknown columns |
| T-03-02 | Tampering | lib/actions/leads.ts | mitigate | assertTransition() called server-side before every DB write — INFR-02 invariant enforced |
| T-03-03 | Tampering | lib/actions/templates.ts | mitigate | zod TemplateSchema validates all fields server-side (name max 100, subject max 200, body max 5000) |
