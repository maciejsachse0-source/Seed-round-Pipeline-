    3. **Create `app/api/export/route.ts`** (per D-07, D-08):
       ```typescript
       import { createClient } from '@/lib/supabase/server'

       const CSV_HEADERS = ['id','name','email','phone','city','source_platform','status','score','created_at']

       export async function GET(request: Request) {
         const { searchParams } = new URL(request.url)
         const format = searchParams.get('format') === 'json' ? 'json' : 'csv'

         const supabase = await createClient()
         const { data } = await supabase
           .from('leads')
           .select('*')
           .in('status', ['interested', 'approved'])
           .order('created_at', { ascending: false })

         const leads = data ?? []

         if (format === 'json') {
           return new Response(JSON.stringify(leads), {
             headers: {
               'Content-Type': 'application/json',
               'Content-Disposition': 'attachment; filename="leads-export.json"',
             },
           })
         }

         // CSV: JSON.stringify each cell value to handle commas, quotes, and formula injection (T-06-05)
         const lines = [
           CSV_HEADERS.join(','),
           ...leads.map(l =>
             CSV_HEADERS.map(h => JSON.stringify((l as Record<string, unknown>)[h] ?? '')).join(',')
           ),
         ]
         return new Response(lines.join('\n'), {
           headers: {
             'Content-Type': 'text/csv; charset=utf-8',
             'Content-Disposition': 'attachment; filename="leads-export.csv"',
           },
         })
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> GET /api/export | Untrusted `format` query parameter |
| Supabase RPC -> analytics page | Data from DB rendered as HTML |
| Lead data -> CSV cells | Untrusted lead data could contain formula injection characters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering | CSV export cell values | mitigate | All CSV cell values wrapped in `JSON.stringify()` which produces double-quoted strings, preventing `=`, `+`, `-`, `@` prefix CSV formula injection. |
| T-06-06 | Tampering | `/api/export?format=` parameter | mitigate | Only `'json'` is accepted as non-default; any other value (including injection attempts) defaults to `'csv'`. No eval, no dynamic import. |
| T-06-07 | Information Disclosure | Export endpoint returns all lead fields | accept | Single-user internal tool with no auth. Export filters to interested/approved status only at DB query level. Acceptable risk for local tool. |
| T-06-08 | Elevation of Privilege | Export endpoint fetching wrong leads | mitigate | `.in('status', ['interested', 'approved'])` filter applied at Supabase query level — cannot be bypassed by URL manipulation. |
</threat_model>
