Create `lib/email/send.ts` exporting:

```typescript
export async function sendColdEmail(lead: Lead, template: EmailTemplate): Promise<{
  success: boolean
  skipReason?: 'suppressed' | 'invalid_mx' | 'cap_reached' | 'no_email'
  emailEventId?: string
  gmailMessageId?: string
}>
```

Implementation flow (in order):
1. Guard: if `!lead.email` return `{ success: false, skipReason: 'no_email' }`
2. Suppression check: `isEmailSuppressed(lead.email)` — return skipReason 'suppressed' if true
3. MX check: `validateMx(lead.email)` — if false, call `addToSuppressionList(lead.email, 'bounce_hard')`, return skipReason 'invalid_mx'
4. Daily cap: `canSendToday()` — return skipReason 'cap_reached' if false
5. Token substitution: call `substituteTokens(template.subject, ...)` and `substituteTokens(template.body, ...)` with `{ name: lead.name ?? '', city: lead.city ?? '', category: lead.categories?.[0] ?? '' }`
6. Generate unsubscribe token: `generateUnsubscribeToken(lead.email, lead.id)`
7. Build unsubscribe URL: `${process.env.NEXT_PUBLIC_APP_URL}/api/unsubscribe?email=${encodeURIComponent(lead.email)}&lead=${lead.id}&token=${unsub_token}`
8. Build tracking pixel URL: `${process.env.NEXT_PUBLIC_APP_URL}/api/track/open/${eventId}` — NOTE: eventId is generated before send as a UUID via `crypto.randomUUID()`
9. Append to HTML body: tracking pixel `<img>` tag + opt-out `<a>` link with Polish text "Nie chcesz otrzymywac wiadomosci? Kliknij tutaj"
10. Send via `getTransporter().sendMail({ from: process.env.GMAIL_SENDER_EMAIL, to: lead.email, subject: substitutedSubject, html: finalBody })`
11. After send: get Gmail message ID. Use `google.gmail('v1')` to call `messages.list({ userId: 'me', q: 'rfc822msgid:' + info.messageId })`, then `messages.get({ id })` to retrieve `historyId` and `threadId`
12. Write email_event to Supabase: `{ id: eventId, lead_id: lead.id, template_id: template.id, sequence_number: 0, sent_at: now, status: 'sent', gmail_message_id, gmail_thread_id, start_history_id: historyId }`
13. Transition lead: `assertTransition(lead.status as LeadStatus, LeadStatus.CONTACTED)`, then update lead status in Supabase
14. Return `{ success: true, emailEventId: eventId, gmailMessageId }`

Wrap steps 10-13 in try/catch. On Nodemailer error: write email_event with status='failed', do NOT transition lead status. Rethrow to let the worker handle logging.

Create `lib/queue/workers/email-worker.ts` following the scrape-worker.ts pattern:

1. Export `registerEmailWorker()`:
   - Get boss via `getBoss()`
   - Call `boss.work('email-send', { localConcurrency: 1 }, async ([job]) => { ... })`
   - CRITICAL: `localConcurrency: 1` prevents race condition on daily cap (see Pitfall 3 in RESEARCH.md)
   - Job data shape: `{ leadId: string, templateId: string }`
   - Inside handler:
     a. Fetch lead from Supabase by leadId — if not found or status not 'approved', log skip and return
     b. Fetch template by templateId — if not found or not is_active, log and return
     c. Call `sendColdEmail(lead, template)`
     d. Log result (success or skipReason)

2. Export `enqueueEmailSend(leadId: string, templateId: string)`:
   - Get boss via `getBoss()`
   - Call `boss.send('email-send', { leadId, templateId }, { retryLimit: 0, startAfter: new Date(Date.now() + SEND_SPACING_MS).toISOString() })`
   - CRITICAL: `retryLimit: 0` — auto-retry could send the same cold email twice
   - `startAfter` enforces 90s minimum spacing between sends (MAIL-06)
