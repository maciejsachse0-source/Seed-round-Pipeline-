<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase CLI → remote DB | CLI uses SUPABASE_ACCESS_TOKEN or interactive login — credentials not stored in repo |
| .env.local → Next.js process | Real credentials in .env.local — must never be committed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | supabase link stores project credentials locally | accept | Credentials stored in ~/.supabase/ (not in repo); acceptable for single-developer tool |
| T-04-02 | Denial of Service | pg-boss connecting to Supabase before schema is pushed | mitigate | Task 1 runs supabase db push BEFORE Task 2 attempts to start the app; instrumentation.ts has try/catch so a startup failure doesn't crash Next.js |
| T-04-03 | Information Disclosure | DATABASE_URL in .env.local contains DB password | mitigate | .gitignore includes .env.local (verified in Plan 03 acceptance criteria) |
</threat_model>
