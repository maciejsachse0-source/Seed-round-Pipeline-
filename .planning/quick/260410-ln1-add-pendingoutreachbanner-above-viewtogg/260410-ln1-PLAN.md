**PendingOutreachBanner.tsx (Server Component):**
```tsx
// components/leads/PendingOutreachBanner.tsx
// Server Component — counts approved-but-not-contacted leads with email
// and renders a CTA banner. Returns null when count === 0.
import { createClient } from '@/lib/supabase/server'
import { PendingOutreachBannerButton } from './PendingOutreachBannerButton'

export async function PendingOutreachBanner() {
  const supabase = await createClient()

  // Count only — head: true avoids fetching rows
  const { count, error } = await supabase
    .from('leads')
    .select('*', { count: 'exact', head: true })
    .eq('status', 'approved')
    .eq('contact_status', 'none')
    .not('email', 'is', null)
    .neq('email', '')

  if (error || !count || count === 0) return null

  return (
    <div className="card p-4 mb-4 border-l-4 border-indigo-400 bg-indigo-50/40">
      <div className="flex items-center justify-between gap-4 flex-wrap">
        <div className="flex items-center gap-3">
          <div className="w-10 h-10 rounded-full bg-indigo-100 flex items-center justify-center shrink-0">
            <span className="text-lg">✉</span>
          </div>
          <div>
            <p className="text-sm font-semibold text-gray-800">
              {count} {count === 1 ? 'lead gotowy' : count < 5 ? 'leady gotowe' : 'leadów gotowych'} do cold email
            </p>
            <p className="text-xs text-gray-500">
              Zatwierdzone leady z adresem email, które jeszcze nie zostały skontaktowane.
            </p>
          </div>
        </div>
        <PendingOutreachBannerButton pendingCount={count} />
      </div>
    </div>
  )
}

export default PendingOutreachBanner
```

Note on Polish pluralization: Polish has three forms — 1 (singular), 2-4 (few), 5+ (many). The check `count === 1` handles singular; `count < 5` roughly covers 2-4; else many. For numbers ending in 12-14 this is slightly wrong (should be "many") but acceptable for a quick task; document with a one-line comment.

**PendingOutreachBannerButton.tsx (Client Component):**
```tsx
// components/leads/PendingOutreachBannerButton.tsx
'use client'
import { useState } from 'react'

interface Props {
  pendingCount: number
}

export function PendingOutreachBannerButton({ pendingCount }: Props) {
  const [sending, setSending] = useState(false)
  const [result, setResult] = useState<{ queued: number; skipped: number; error?: string } | null>(null)

  async function handleClick() {
    setSending(true)
    try {
      const res = await fetch('/api/batch-send', { method: 'POST' })
      const data = await res.json()
      if (res.ok) {
        setResult(data)
      } else {
        setResult({ queued: 0, skipped: 0, error: data.error || 'Błąd wysyłki' })
      }
    } catch {
      setResult({ queued: 0, skipped: 0, error: 'Nie udało się połączyć z API' })
    } finally {
      setSending(false)
    }
  }

  if (result && !result.error) {
    return (
      <div className="text-sm text-emerald-700 font-medium flex items-center gap-2">
        <svg className="w-4 h-4" fill="none" viewBox="0 0 24 24" stroke="currentColor" strokeWidth={2.5}>
          <path strokeLinecap="round" strokeLinejoin="round" d="M4.5 12.75l6 6 9-13.5" />
        </svg>
        Zakolejkowano {result.queued} {result.queued === 1 ? 'email' : 'emaili'}
        {result.skipped > 0 && <span className="text-xs text-gray-400">({result.skipped} pominięto)</span>}
      </div>
    )
  }

  if (result?.error) {
    return (
      <div className="text-sm text-red-600 font-medium max-w-xs text-right">
        {result.error}
      </div>
    )
  }

  return (
    <button
      type="button"
      onClick={handleClick}
      disabled={sending}
      className="btn-primary flex items-center justify-center gap-2 disabled:opacity-50 shrink-0"
    >
      {sending ? (
        <>
          <svg className="animate-spin h-4 w-4" viewBox="0 0 24 24" fill="none">
            <circle className="opacity-25" cx="12" cy="12" r="10" stroke="currentColor" strokeWidth="4" />
            <path className="opacity-75" fill="currentColor" d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4z" />
          </svg>
          Kolejkuję...
        </>
      ) : (
        `Wyślij cold email (${pendingCount})`
      )}
    </button>
  )
}
```

**Things to avoid and WHY:**
- DO NOT import SessionComplete from LeadCard.tsx and reuse it. SessionComplete is designed for a full-column "session done" celebration layout with padding py-16 and max-w-md — wrong shape for a horizontal banner.
- DO NOT use the service_role client (lib/supabase/service.ts). The user is authenticated via cookies; use @/lib/supabase/server which respects RLS. Service role in a request-scoped component is a threat vector (T-03-01 style).
- DO NOT call the count query with `.select('id')` then `.length` — that fetches rows. Use `{ count: 'exact', head: true }` to get count with zero rows transferred.
- DO NOT add `export const dynamic = 'force-dynamic'` here. The query uses cookies (via createClient), so Next.js will automatically mark the parent route dynamic. No manual flag needed.
- DO NOT handle the "no sequence step 0 template" error client-side by hiding the banner — the user still has pending leads; surface the server error message from /api/batch-send in the error state so the operator knows to fix the sequence.
- DO NOT use `<img>` for any icon; use inline SVG (matches LeadCard.tsx pattern).
